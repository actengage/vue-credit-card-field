vue-credit-card-field

Overview

This component is meant to be a standalone Vue component that mimics the UX of the Stripe.js credit card field without using an iframe AND is compatible with Boostrap v4. This component is no way meant to replace Stripe.js if you are already using it.

Basic Usage

The field works like most input fields and will update the model reactively. The card is represent as an object with various properties associated with the input.

{{ card }}

Error Handling

You will most likely be do some processing on the backend to send the information gathered to the server that will inevitable result in an error of some kind. Errors can be passed in two ways: string or object.

String

Object

Notice, if you pass an object of errors, the name attribute is required to mark which key/value pair to use. An error may be a string or an array of strings.

Events Callbacks

Use the event callbacks to generate a card token, add additional error handling, or to send the card data for processing.

Card Change Event: {{ lastChangeEvent }}
Card Invalid Event: {{ lastInvalidEvent }}
